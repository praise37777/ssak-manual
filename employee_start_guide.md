# 🚀 워드프레스 자동화 세팅 — 직원용 시작 가이드

안녕하세요! 이 문서는 **새 홈페이지를 자동화 시스템에 연결하는 작업**을 하실 때 사용하는 안내서입니다.

---

## 📋 작업 시작 전 준비물 체크리스트

작업을 시작하기 전에 아래 항목을 모두 준비해주세요:

- [ ] **Claude 접속 가능한 브라우저** (크롬 추천)
- [ ] **새 홈페이지 관리자 계정** (아이디/비밀번호)
- [ ] **n8n 접속 가능한 계정**
- [ ] **Make.com 접속 가능한 계정**
- [ ] **구글 계정** (대표님이 알려주신 공용 계정)
- [ ] **호스팅 관리자 계정** (HostGator 등)
- [ ] **이 가이드에서 알려주는 두 파일**
  - `sonnet_context.md` — Claude에게 업로드할 시스템 설명 파일
  - 매뉴얼 링크: `https://praise37777.github.io/ssak-manual/`

---

## 🎯 작업 순서 (매번 이렇게 시작하세요)

### 1단계 — Claude 새 채팅 시작

1. Claude 앱 또는 `claude.ai` 접속
2. **"New chat"** 또는 새 대화 시작
3. **모델을 Sonnet으로 선택** (빠르고 정확함)

### 2단계 — 컨텍스트 파일 업로드

Claude에게 `sonnet_context.md` 파일을 **먼저** 업로드합니다.

**첫 메시지 예시:**
```
안녕! 싹싹컴퍼니 워드프레스 자동화 작업을 할 거야.
업로드한 파일은 이 작업의 전체 시스템 설명서야.
이 파일을 읽고 나를 도와줘.

오늘 작업할 사이트: [새 홈페이지 주소]
지금 어느 단계부터 시작하면 될지 알려줘.
```

### 3단계 — 매뉴얼 열기

새 탭에서 매뉴얼 열기:
👉 **https://praise37777.github.io/ssak-manual/**

이 매뉴얼은 10단계로 나뉘어 있어요.

### 4단계 — 단계별로 진행하기

매뉴얼 1단계부터 순서대로 작업합니다. 각 단계 끝에 **체크포인트**가 있어요.

---

## 📸 체크포인트 — 매 단계마다 이렇게 보고하세요

각 단계가 끝나면 Claude에게 아래 형식으로 보고하세요:

### ✅ 체크포인트 보고 템플릿
```
[단계 번호] 완료했어요.
[해당 단계의 스크린샷 첨부]
다음으로 넘어가도 될까요?
```

### 단계별 필수 스크린샷

| 단계 | 제목 | 찍어야 할 스크린샷 |
|------|------|------|
| **1단계** | 구글 시트 복사 | 새 시트가 열린 상태의 **주소창 전체**(ID 확인용) |
| **2단계** | n8n 폴더 확인 | 테스트 폴더의 **5개 워크플로 목록 전체** |
| **3단계** | 서브워크플로 교체 | `자동화 copy`의 **3개 노드 확대 스크린샷** (Workflow 항목에 copy로 표시되어야 함) |
| **4단계** | Credential 재연결 | 각 워크플로의 **전체 캔버스** (⚠️ 경고가 다 사라진 모습) |
| **5단계** | Make URL 교체 | `Create a Post to Make.com` 노드의 **URL 칸 확대** |
| **6단계** | Make 시나리오 복제 | 복제된 시나리오의 **전체 뷰 + 웹훅 URL 칸** |
| **7단계** | WordPress 교체 | WordPress 노드 7개 각각의 **Connection 설정 창** |
| **8단계** | 구글 시트 교체 | Google Sheets 노드 2개의 **Spreadsheet ID 칸** |
| **9단계** | 전체 활성화 | n8n 5개 모두 **"Published" 상태 + Make 토글 ON** |
| **10단계** | 테스트 | 테스트 실행 결과 **전체 워크플로 초록색 상태** |

## 🆘 막혔을 때 — Claude에게 물어보는 법

### 잘 질문하는 법

#### ❌ 안 좋은 질문
> "안 돼요"
>
> "오류 떴어요"
>
> "여기 뭐해요?"

#### ✅ 좋은 질문
> "매뉴얼 3단계에서 '발행 여부 설정' 노드를 클릭했는데 우측 패널에 Workflow 항목이 안 보여요. 스크린샷 첨부해요."
>
> "4단계 진행 중인데 이 노드에 빨간색 오류가 떴어요. 어떻게 해야 하나요?"
>
> "7단계에서 Connection 드롭다운에 기존 계정이 안 보여요. 새로 만들어야 하나요?"

### 오류가 떴을 때

1. **오류 메시지 전체 스크린샷** 찍기
2. **어느 단계, 어느 노드**에서 발생했는지 명시
3. **직전에 무엇을 했는지** 설명
4. Claude에게 전송

예시:
```
매뉴얼 7단계 진행 중이에요.
Make 시나리오에서 "WordPress — Create a Post" 노드 클릭했는데
"Connection failed" 오류가 떴어요.
방금 새 Connection을 만들었고, API Key는 Make Connector에서 복사했어요.
스크린샷 첨부합니다.
```

## ⚠️ 절대 하지 말아야 할 것

### 🚫 금지 사항
1. **원본 워크플로 수정 금지** — 반드시 `copy` 버전만 수정
2. **원본 Make 시나리오 수정 금지** — 반드시 복제본만 수정
3. **wp-config.php 직접 수정 금지** — 반드시 대표님께 먼저 보고
4. **비밀번호, API Key를 채팅창에 그대로 붙여넣기 금지** — 유출 위험
5. **모르는 플러그인 함부로 설치 금지** — 보안 문제 발생 가능

### 🔐 민감 정보 취급
- DB 비밀번호, 관리자 비밀번호 같은 건 **대표님과 직접 확인**
- 파일 내용을 공유할 때 비밀번호 부분은 **가리고 공유**
- 의심스러운 코드 발견 시 (`@eval`, `base64_decode` 등) **즉시 대표님께 보고**
